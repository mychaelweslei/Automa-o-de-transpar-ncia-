Este projeto consiste em um script de automação desenvolvido em Python utilizando a biblioteca PyAutoGUI. Seu principal objetivo é simular o comportamento de um usuário real para navegar, interagir e validar a acessibilidade de documentos e relatórios críticos no portal de Transparência do município.1. Objetivo PrincipalValidar a funcionalidade e a acessibilidade dos principais módulos do Portal de Transparência (Módulos COVID-19, Licitações e Educação), garantindo que os links para documentos (PDFs) e relatórios estejam operacionais.Este script é uma ferramenta fundamental para o QA (Quality Assurance) de rotina e para a verificação da integridade dos dados expostos ao público.2. Funcionalidades AutomatizadasO script executa uma sequência completa de ações, dividida em módulos lógicos. A URL de destino é: https://transparencia.pmspa.rj.gov.br/.Módulo AutomatizadoAções RealizadasInicializaçãoAbre o Menu Iniciar do Windows, pesquisa e inicia o Google Chrome, maximiza a janela e navega para a URL de Transparência.Módulo COVID-19Navega por três ícones de conteúdo, abre e fecha um arquivo PDF e utiliza comandos de navegação de retorno e início (Home).Módulo LicitaçõesInterage com três diferentes ícones do módulo. Em um dos ícones, simula o download e a abertura de um relatório, garantindo que o fluxo de download esteja funcional.Módulo EducaçãoRola a tela (scroll down), clica em dois ícones e simula o download e a abertura de um arquivo baixado (o relatório).FinalizaçãoRetorna à página inicial do portal.3. DependênciasO projeto requer as seguintes bibliotecas Python:pyautogui (Para controle de mouse e teclado)time (Para gerenciar pausas e tempos de espera)Instalação:Bashpip install pyautogui
4. Execução e SetupRequisitos de Ambiente:Python: Versão 3.x instalada.Configuração de Tela: As coordenadas de cliques são fixas. O script só funcionará de forma consistente se a resolução da tela e a posição da janela do navegador (maximizado) forem as mesmas do ambiente de desenvolvimento.Nota Crítica (Limitação do PyAutoGUI):O PyAutoGUI baseia-se em coordenadas fixas. Mudanças na resolução da tela, tamanho da barra de tarefas ou posição da janela causarão falha na automação. Este projeto serve como um MVP (Produto Mínimo Viável) e ponto de partida para a implementação de um framework mais robusto (como Selenium) no futuro.5. Próximos Passos (Evolução do Projeto)A próxima fase de desenvolvimento deve focar na criação de uma Arquitetura de Teste Inteligente que:Contorne barreiras de segurança (como o reCAPTCHA).Implemente lógica de decisão (para tratar erros e ler PDFs).Substitua as coordenadas fixas por seletores de elementos (Web).
